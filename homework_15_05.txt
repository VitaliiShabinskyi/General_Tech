1. Написать все наше пройденные ограничение, и написать своими словами кто за что отвечает, представьте вы это объясните своему другу

NOT NULL - Ячейка не может иметь нулевое значение.
UNIQUE - Все значения в колонке будут уникальными.
PRIMARY KEY - Определяет уникальность строки. Используется в комбинации с NOT NULL и UNIQUE. В таблице используется только 1 уникальный ключ.
FOREIGN KEY - Внешний ключ нужен для того, чтобы связать две разные SQL-таблицы между собой.
CHECK - Гарантирует, что все значения в столбце удовлетворяют определенному условию
INDEX - Используется для быстрого создания и извлечения данных из базы данных.

DEFAULT - Задает значение по умолчанию для столбца, если значение не указано при добавлении данных в таблицу.

CREATE TABLE products (

3. Напишите своими словами что такое проверка в MYSQL, и как она пишется в Инициализация таблицы

CHECK используется для ограничения диапазона значений, который может быть помещен в столбец.
Если вы определяете контрольное ограничение для одного столбца, оно допускает только определенные значения для этого столбца.
Если вы определяете контрольное ограничение для таблицы, оно может ограничивать значения в определенных столбцах на основе значений в других столбцах строки.

4*. Напишите чем отличается union от union all, с примером

UNION объединяет 2 и более результата SELECT с удалением дубликатов
UNION ALL - объединяет 2 и более результата SELECT без удаления дубликатов

SELECT 
    FirstName, LastName
FROM
    Customers 
UNION SELECT 
    FirstName, LastName
FROM
    Employees;


SELECT 
    FirstName, LastName
FROM
    Customers 
UNION ALL SELECT 
    FirstName, LastName
FROM
    Employees
ORDER BY FirstName;
